# Emm42_V5.0 / ZDT_X42_V1.2 闭环驱动器协议与功能说明

本文第 1–10 节描述张大头 Emm42_V5.0（采用 ZDT_X42_V1.2 硬件框架）驱动器本身的功能、接口和自定义通讯协议；第 11–12 节补充仓库中的 Klipper 监控与在线 PID 调参封装。

本文依据仓库中的 [`Emm.pdf`](../Emm.pdf)（Rev1.3，2023-11-20）整理。手册没有定义的内容会明确标为“待验证”；“待验证”不应当作为协议实现的确定依据。

## 1. 产品能力与工作模式

### 1.1 核心能力

Emm42_V5.0 的主要能力包括：

- FOC 矢量闭环控制，内部包含力矩环、速度环和位置环；
- 板载 16384 线磁编码器和相电流传感器；编码器经过内部线性化校准后，一圈使用 `0..65535` 的数值表示；
- 供电规格在产品参数中写为 `7–32 V`，最大工作电流为 `0–3000 mA`，最高转速因电机而异，手册给出 `3000 RPM+` 的上限描述；
- 脉冲控制，以及 TTL 串口、RS232、RS485、CAN 通讯控制；脉冲和通讯控制可以混合使用；
- 速度模式和位置模式；位置命令支持相对位置和绝对位置，支持曲线加减速和立即停止；
- 0.1 RPM 输入分辨率选项、速度模式参数上电自动运行、En 引脚启停；
- 编码器自校准、相电阻/相电感辨识、线序错误识别、欠压提示、堵转检测与堵转保护；
- 单圈就近回零、单圈方向回零、多圈无限位碰撞回零、多圈有限位开关回零；
- 通讯控制下的位置到达标志和到位返回；脉冲控制下的报警输出和堵转保护复位。

### 1.2 脉冲端口工作模式

`P_Pul` 菜单决定脉冲端口的用途：

| 菜单值 | 含义 |
| --- | --- |
| `PUL_OFF` | 关闭脉冲输入。 |
| `PUL_OPEN` | 开环模式，不依赖编码器工作；手册给出的最高转速约为 `200–400 RPM`。 |
| `PUL_FOC` | FOC 矢量闭环模式，电流实时调节；最高转速约为 `3000 RPM+`，取决于电机。 |
| `ESI_RCO` | 将脉冲相关端口复用为限位输入和到位输出。 |

通讯控制并不等同于脉冲输入：开环/闭环模式都可以接受通讯控制，但电机必须处于可使能状态，且不能处于堵转保护状态。

### 1.3 通讯端口复用模式

`P_Serial` 菜单的意义如下：

| 菜单值 | 含义 |
| --- | --- |
| `RxTx_OFF` | 关闭通讯端口。 |
| `ESI_ALO` | 将端口复用为限位输入和报警输出。 |
| `UART_FUN` | 将 `R/A/H`、`T/B/L` 复用为 TTL 串口、RS232 或 RS485 通讯。 |
| `CAN1_MAP` | 将 `R/A/H`、`T/B/L` 复用为 CAN 通讯。 |

TTL、RS232、RS485 使用相同的逻辑报文格式；物理电气层需要使用对应的 ZDT 模块。CAN 使用 `ZDT_CAN` 模块和扩展帧，详见第 2、3 节。

### 1.4 电流、堵转和到位

- `Ma` 是开环模式运行时的相电流，通讯可设置范围为 `0–3000 mA`。
- `Ma_Limit` 是 FOC 闭环模式发生堵转时允许的最大相电流，通讯可设置范围为 `0–3000 mA`。
- `Op_Limit` 是闭环模式的最大输出电压，手册菜单单位为 mV；它可以近似限制最大电流和转速。
- `Clog_Pro` 使能后，满足堵转条件时驱动器会自动关闭；即使未使能，LED 也会提示堵转。堵转判定条件是三个条件同时满足：

  ```text
  实际转速 < 堵转检测转速阈值
  实际相电流 > 堵转检测相电流阈值
  持续时间 > 堵转检测时间阈值
  ```

- `P_Serial=ESI_ALO` 时，堵转或欠压等异常会使 `T/B/L` 输出高电平；将 `R/A/H` 接地可复位堵转保护。通讯控制下也可以使用 `0x0E` 命令解除堵转保护。
- 位置到达由目标位置和传感器实时位置的差值与“位置到达窗口”比较产生；到位时置位电机状态标志的 bit1，并且可以按 `Response` 菜单返回 `A FD 9F C`。手册在不同章节给出默认窗口 `0.1°` 和 `0.3°` 两种说法，不能据此假设默认值，应读取实际驱动配置。

### 1.5 编码器校准与回零能力

编码器校准用于线性化插值和电角度对齐。校准前应选择正确的电机类型并让电机空载；驱动板从电机拆下后，磁铁与编码器的相对位置可能改变，手册要求恢复出厂设置后重新空载校准。通讯触发校准使用 `0x06`。

回零模式及编码如下：

| 编码 | 模式 | 说明 |
| --- | --- | --- |
| `0x00` | `Nearest` | 单圈就近回零。 |
| `0x01` | `Dir` | 单圈方向回零，方向由回零方向参数决定。 |
| `0x02` | `Senless` | 多圈无限位碰撞回零；利用转速下降、相电流升高和持续时间判断碰撞。 |
| `0x03` | `EndStop` | 多圈有限位开关回零。 |

## 2. CAN 硬件与菜单配置

### 2.1 必要菜单项

使用 CAN 时至少需要确认以下菜单：

| 菜单 | 设置/编码 | 协议意义 |
| --- | --- | --- |
| `P_Serial` | `CAN1_MAP` | 将通讯端口切换到 CAN。不是 `CAN1_MAP` 时，CAN 控制无效。 |
| `CAN_Baud` | `10000、20000、50000、83333、100000、125000、250000、500000、800000、1000000` | CAN 比特率，默认 `500000 bit/s`（500 kbit/s）。`0x42` 配置字段保存的是菜单顺序索引；手册示例 `0x07=500000`。 |
| `ID_Addr` | `1–255`；`0` 为广播 | 逻辑节点地址，同时参与 CAN 扩展帧 ID。菜单可直接选择有限范围，其他地址可通过通讯修改。 |
| `Checksum` | `0x6B`、`XOR`、`CRC-8`；菜单另列 `Modbus` | 本文只覆盖 `0x6B`、XOR 和 CRC-8 自定义协议；Modbus-RTU 属于手册另行说明的协议，不在本文范围。 |
| `Response` | `None`、`Receive`、`Reached`、`Both`、`Other` | 控制动作命令是否返回确认，以及位置命令是否返回到位消息。默认是 `Receive`。 |

`Response` 的语义为：

| 选项 | 行为 |
| --- | --- |
| `None` | 不返回收到确认，也不返回位置到位消息。 |
| `Receive` | 返回收到确认；默认不额外返回到位消息。 |
| `Reached` | 位置模式只返回到位消息。 |
| `Both` | 返回收到确认，并在位置到达后返回到位消息。 |
| `Other` | 位置模式只返回到位消息，其他控制动作返回收到确认。 |

### 2.2 ZDT_CAN 模块与接线

手册 CAN 接线图给出的硬件要求如下：

- 每个 `ZDT_X42_V1.2` 驱动器的插针上插入一个 `ZDT_CAN` 模块；模块不能带电插拔。
- 驱动器 `R/A/H` 接模块/总线的 `CAN_H`（手册图中标为 `H`）。
- 驱动器 `T/B/L` 接模块/总线的 `CAN_L`（手册图中标为 `L`）。
- `V+` 和 `Gnd` 按驱动器供电要求接电源；CAN 多机图同时引出 `Gnd`、`CAN_H`、`CAN_L`，节点并联到同一条总线。
- 手册 CAN 接线图标注 `12–32 V、3 A 以上`供电；产品参数章节又写成 `7–32 V`。实际电源应按驱动器、电机和负载的额定要求确认，不能只依据一个章节的范围。

CAN 总线的端接要求不能从 `Emm.pdf` 的文字或图中得到完整型号专属结论：手册没有写出端接电阻阻值、模块内是否默认接入终端、也没有说明终端跳线位置。物理 CAN 总线通常应只在总线两端各设置一个 `120 Ω` 终端，节点中间不重复加终端；是否由 `ZDT_CAN` 模块内置或需要外接，必须以模块实物标识和断电测量确认。不要把“每个节点都加 120 Ω”当成该驱动器的确定接线规则。

## 3. 通讯帧格式

### 3.1 串口/RS232/RS485 的逻辑报文

逻辑报文统一为：

```text
地址 A | 功能码 F | 指令数据 D | 校验字节 C
```

发送和接收都使用同一排列。本文所有“串口逻辑报文”示例的长度均以实际二进制字节计数，不包括空格和字符串表示的 `0x`。

地址规则：

- `A=0x01..0xFF` 是单播地址；默认地址为 `0x01`。
- `A=0x00` 是广播地址。广播命令由所有驱动器执行，但只有地址为 `0x01` 的驱动器返回确认；其他地址不回复。
- 485 理论上最多挂载 256 个从机；CAN 理论上最多连接 110 个节点。这是手册给出的理论数字，不等于所有布线、负载和比特率下的保证值。

### 3.2 CAN 扩展帧 ID 与 payload

CAN 帧固定使用扩展帧。扩展帧 ID 定义为：

```text
CAN_ID = (ID_Addr << 8) | packet_number
```

其中 `packet_number` 是低 8 位的分包号，从 `0x00` 开始。地址为 `1` 的第 0 包的 ID 是 `0x0100`；工具显示为 29 位扩展 ID 时可能显示成 `0x00000100`。

CAN 线上的 payload 与串口逻辑报文不是同一串字节：

| 传输层 | 地址位置 | 小帧内容 |
| --- | --- | --- |
| 串口/RS232/RS485 | 报文第 0 字节 | `A F D C` |
| CAN | 扩展帧 ID 的高字节 | payload 为 `F D C`，地址不再重复放入 payload |

因此，逻辑报文 `01 24 6B` 在 CAN 上应表现为：`CAN_ID=0x0100`、payload=`24 6B`。手册 6.4 节的小帧抓帧示例和 CAN 实际抓帧均采用这种“地址在帧 ID、payload 不含地址”的形态；仍应注意这只是本协议的 CAN 线格式，不能把它反向当作串口格式。

CAN payload 的单帧上限是 8 字节。手册把命令称为“小于 8 字节命令”，实际使用时应按 CAN payload/DLC 是否超过 8 字节判断；地址从 payload 移出后，CAN payload 通常比串口逻辑报文少 1 字节。

### 3.3 大于 8 字节命令的分包

手册对位置命令给出的分包规则是：

1. 第 0 包使用 `packet_number=0`，后续包依次使用 `1、2、...`。
2. 每个 CAN payload 不超过 8 字节。
3. 每个包都带功能码；功能码不是只在第 0 包出现。
4. 地址只放在 CAN ID，不放在 payload。
5. 校验字节放在最后一个包的末尾。

手册示例的串口逻辑命令是：

```text
01 FD 01 0F A0 00 00 01 FA 00 00 00 6B   # 13 字节
```

对应 CAN 发送为：

```text
CAN_ID=0x0100  payload=FD 01 0F A0 00 00 01 FA   # 第 0 包，8 字节
CAN_ID=0x0101  payload=FD 00 00 00 6B             # 第 1 包，5 字节
```

第二包再次带 `FD`，所以 CAN 线上分包后的字节总数不等于简单地删除地址后再切片。手册没有规定分包重传、乱序、超时、长响应的最大包数或接收端的重组超时；这些行为均需实测确认。特别是 `0x21` PID 读取的响应超过 8 字节，手册没有给出它的 CAN 长响应抓帧示例，不能据此推导出未公开的长响应规则。

### 3.4 校验字节

本文的完整示例使用默认固定校验：

```text
C = 0x6B
```

手册还定义了两种自定义计算选项：

- XOR：将校验前的全部逻辑字节异或。手册示例 `01 06 45 42` 中，`42 = 01 ^ 06 ^ 45`。
- CRC-8：将校验前的全部逻辑字节进行 CRC-8 计算。

手册没有完整给出 CRC-8 的多项式、初始值、输入/输出反射和最终异或值，因此本文不指定任何 CRC 算法参数。“CRC-8 选项存在”是确定信息，“使用某一个常见 CRC-8 参数”不是确定信息，必须通过设备实测确认。

CAN 下地址由帧 ID 承载，手册也没有明确 XOR/CRC 是否把地址字节折算进校验。固定 `0x6B` 不受这个歧义影响；XOR/CRC 在 CAN 下属于待验证行为。

## 4. 数据编码规则

### 4.1 整数、符号和存储标志

- 多字节整数均按大端序发送：高字节在前。例如 `1500` 为 `0x05DC`，发送 `05 DC`；`10000` 为 `0x00002710`，发送 `00 00 27 10`。
- 读取位置、转速、脉冲等带符号量时，符号独立占一个字节：`0x00=正数`，`0x01=负数`。数值字段本身仍按无符号大端幅值解释，不是补码字段。
- 修改命令中的“是否存储/存储标志”通常为一字节：`0x00` 表示本次不保存，`0x01` 表示保存到芯片。断电后的行为以具体命令和驱动器固件为准。
- “命令状态”通常为一字节：`0x02` 表示命令正确；`0xE2` 表示手册列出的条件不满足；`0xEE` 出现在通用错误响应中。

### 4.2 编码器、位置和角度

经过线性化校准的编码器值是 16 位，一圈使用 `0..65535`：

```text
encoder_angle_deg = encoder_value * 360 / 65536
```

位置、实时位置和位置误差的幅值字段按每圈 65536 个角度计数换算：

```text
angle_deg = sign * raw_value * 360 / 65536
```

其中 `sign` 为 `+1` 或 `-1`。多圈位置可超过一圈，因此目标位置/实时位置/位置误差字段使用 32 位幅值；手册示例 `0x00010000` 且符号为负表示 `-360.0°`。位置误差示例 `0x00000008` 且符号为负表示约 `-0.0439453125°`。

输入脉冲数是独立的带符号 32 位计数。手册以 1.8° 电机、16 细分为例：一圈是 3200 个脉冲；32 细分时一圈是 6400 个脉冲。

### 4.3 速度、加速度和电气量

| 字段 | 编码/单位 |
| --- | --- |
| 速度 | 16 位无符号大端，单位 RPM；方向单独用 `0x00=CW`、`0x01=CCW` 表示。 |
| 速度输入缩放 | `S_Vel_IS` 使能后，通讯输入值缩小 10 倍，例如发送 `1` 实际以 `0.1 RPM`运行；该选项同时影响速度模式、位置模式和上电速度参数。具体固件对小数输入的边界需实测。 |
| 加速度档位 | 1 字节，`0..255`。`0` 表示不使用曲线加减速；手册给出 `t2-t1=(256-acc)*50 µs`、`Vt2=Vt1+1 RPM`。 |
| 总线电压 | 16 位无符号大端，单位 mV。读取值是经过输入反接二极管后的电压，手册提示会有压降。 |
| 相电流 | 16 位无符号大端，单位 mA。 |
| 相电阻 | 16 位无符号大端，单位 mΩ。 |
| 相电感 | 16 位无符号大端，单位 µH。 |
| 回零速度/堵转检测速度 | 16 位无符号大端，单位 RPM。 |
| 回零/堵转检测电流 | 16 位无符号大端，单位 mA。 |
| 回零/堵转检测时间 | 16 或 32 位，具体命令按字段表，单位 ms。 |

### 4.4 状态标志位

回零状态字节（`0x3B`）和 `0x43` 中的“就绪状态标志”使用相同位定义：

| 位 | 掩码 | 含义 |
| --- | --- | --- |
| bit0 | `0x01` | 编码器就绪。 |
| bit1 | `0x02` | 校准表就绪。 |
| bit2 | `0x04` | 正在回零。 |
| bit3 | `0x08` | 回零失败。 |

电机状态字节（`0x3A` 和 `0x43` 中的“电机状态标志”）定义为：

| 位 | 掩码 | 含义 |
| --- | --- | --- |
| bit0 | `0x01` | 电机使能。 |
| bit1 | `0x02` | 电机到位。 |
| bit2 | `0x04` | 电机堵转。 |
| bit3 | `0x08` | 堵转保护已触发。 |

## 5. 通用响应与异常

### 5.1 正常、条件失败和通用错误

对有状态响应的命令，串口逻辑响应通常为：

```text
A F 02 C       # 命令正确
A F E2 C       # 命令格式正确，但执行条件不满足
A 00 EE C      # 通用错误/错误命令
```

在默认固定校验下，地址 `0x01` 的典型字节分别是 `01 F 02 6B`、`01 F E2 6B` 和 `01 00 EE 6B`。CAN command-only payload 中地址移入帧 ID，因此对应于 `F 02 6B`、`F E2 6B` 和 `00 EE 6B`；响应地址由扩展帧 ID 表示。

`E2` 只适用于手册明确列出条件不满足的命令，例如未使能、堵转保护、未处于回零状态等。没有列出 `E2` 条件的读取命令，不能自行假设会返回 `E2`。

### 5.2 如何区分无响应

- 校验错误：手册没有规定驱动器应返回哪一种错误帧；收到无响应时不能把它确定解释为 `EE` 或 `E2`。
- 地址不匹配：单播命令不是本机地址时，手册未定义显式“地址错误”响应；通常应先按寻址规则检查是否应该无响应。
- 广播：所有节点执行，但只有地址 `0x01` 回复；无其他地址的回复并不表示执行失败。
- 模式/执行条件错误：如果收到了 `A F E2 C`，表示驱动器识别了功能码，但当前条件不满足；具体条件按命令条目解释。
- 格式/功能码/数据错误：手册示例使用 `A 00 EE C`。CAN 下应先确认帧 ID、payload 是否省略地址，以及分包号和功能码是否正确。
- 完全无帧：还可能由 `CAN1_MAP`、CAN 比特率、扩展帧类型、模块、H/L 接线、端接或地址错误引起；手册没有定义一个可以仅凭“无响应”确定根因的协议码。

## 6. 控制命令

下列命令均先以串口逻辑格式给出。`A` 是地址，`C` 是校验字节，示例默认 `C=6B`。对应 CAN 线格式请按第 3.2、3.3 节移除地址并放入扩展帧 ID。

### 6.1 控制动作命令（手册 6.3.1）

#### 6.1.1 电机使能控制（`0xF3`）

- 请求：`A F3 AB EN SY C`（6 字节）。`EN=00` 不使能，`EN=01` 使能；`SY=00` 不等待多机同步，`SY=01` 等待同步触发。
- 响应：`A F3 ST C`（4 字节）。`ST=02` 成功，`ST=E2` 条件不满足。
- 字段：`AB` 为固定子命令 1 字节；`EN`、`SY` 各 1 字节；无物理单位。
- 示例：`01 F3 AB 01 00 6B`（6 字节）→ `01 F3 02 6B`（4 字节）。
- 错误：错误命令 `01 00 EE 6B`；条件不满足 `01 F3 E2 6B`。

#### 6.1.2 速度模式控制（`0xF6`）

- 请求：`A F6 DIR RPM[2] ACC SY C`（8 字节）。
- 响应：`A F6 ST C`（4 字节）。
- 字段：`DIR=00` 为 CW、`DIR=01` 为 CCW；`RPM` 为大端 16 位 RPM；`ACC` 为 1 字节加速度档位；`SY` 为多机同步标志。
- 示例：`01 F6 01 05 DC 0A 00 6B`（8 字节）表示 CCW、1500 RPM、加速度 10、不同步，返回 `01 F6 02 6B`。
- 错误：堵转保护或电机未使能时条件失败 `01 F6 E2 6B`；错误命令 `01 00 EE 6B`。

#### 6.1.3 位置模式控制（`0xFD`）

- 请求：`A FD DIR RPM[2] ACC PULSES[4] RELABS SY C`（13 字节）。
- 响应：`A FD ST C`（4 字节）。
- 字段：`DIR` 为方向；`RPM` 为 RPM；`ACC` 为加速度档位；`PULSES` 为大端 32 位脉冲数；`RELABS=00` 为相对位置，`01` 为绝对位置；`SY` 为多机同步标志。
- 示例：`01 FD 01 05 DC 00 00 00 7D 00 00 00 6B`（13 字节）表示 CCW、1500 RPM、无曲线加减速、32000 脉冲、相对位置、不同步，返回 `01 FD 02 6B`。
- 错误：堵转保护或电机未使能时条件失败 `01 FD E2 6B`；错误命令 `01 00 EE 6B`。

#### 6.1.4 立即停止（`0xFE`）

- 请求：`A FE 98 SY C`（5 字节）。`98` 为固定子命令；`SY` 为多机同步标志。
- 响应：`A FE ST C`（4 字节）。
- 字段：无单位；用于速度模式和位置模式的立即停止/紧急刹车。
- 示例：`01 FE 98 00 6B`（5 字节）→ `01 FE 02 6B`（4 字节）。
- 错误：堵转保护或电机未使能时 `01 FE E2 6B`；错误命令 `01 00 EE 6B`。

#### 6.1.5 多机同步运动（`0xFF`）

- 请求：`A FF 66 C`（4 字节）。`66` 为固定子命令。
- 响应：`A FF ST C`（4 字节）。
- 字段：无数据字段；触发此前以 `SY=01` 暂存的多台电机运动。
- 示例：`01 FF 66 6B`（4 字节）→ `01 FF 02 6B`（4 字节）。
- 错误：条件不满足时 `01 FF E2 6B`；错误命令 `01 00 EE 6B`。

### 6.2 原点回零命令（手册 6.3.2）

#### 6.2.1 设置单圈回零零点（`0x93`）

- 请求：`A 93 88 STORE C`（5 字节）。`88` 为固定子命令，`STORE` 为存储标志。
- 响应：`A 93 ST C`（4 字节）。
- 字段：`STORE` 无单位；发送前电机轴应处于希望的单圈零点位置。
- 示例：`01 93 88 01 6B`（5 字节）→ `01 93 02 6B`（4 字节）。
- 错误：错误命令 `01 00 EE 6B`；手册未列出该命令的独立 `E2` 条件。

#### 6.2.2 触发回零（`0x9A`）

- 请求：`A 9A HOME_MODE SY C`（5 字节）。
- 响应：`A 9A ST C`（4 字节）。
- 字段：`HOME_MODE=00` 单圈就近、`01` 单圈方向、`02` 多圈无限位碰撞、`03` 多圈有限位开关；`SY` 为多机同步标志。
- 示例：`01 9A 00 00 6B`（5 字节）→ `01 9A 02 6B`（4 字节）。
- 错误：堵转保护、电机未使能或单圈零点无效时 `01 9A E2 6B`；错误命令 `01 00 EE 6B`。

#### 6.2.3 强制中断回零（`0x9C`）

- 请求：`A 9C 48 C`（4 字节）。`48` 为固定子命令。
- 响应：`A 9C ST C`（4 字节）。
- 字段：无单位；用于正在执行的回零过程。
- 示例：`01 9C 48 6B`（4 字节）→ `01 9C 02 6B`（4 字节）。
- 错误：当前没有回零操作时 `01 9C E2 6B`；错误命令 `01 00 EE 6B`。

#### 6.2.4 读取回零参数（`0x22`）

- 请求：`A 22 C`（3 字节）。
- 响应：`A 22 MODE DIR VEL[2] TIMEOUT[4] SL_RPM[2] SL_MA[2] SL_MS[2] POWER_ON C`（18 字节）。
- 字段：`MODE` 1 字节；`DIR=00` 为 CW、`01` 为 CCW；`VEL` 单位 RPM；`TIMEOUT` 单位 ms；`SL_RPM` 碰撞检测转速单位 RPM；`SL_MA` 碰撞检测电流单位 mA；`SL_MS` 碰撞检测时间单位 ms；`POWER_ON=00/01` 表示是否上电自动回零。
- 示例：`01 22 6B`（3 字节）→ `01 22 00 00 00 1E 00 00 27 10 01 2C 03 20 00 3C 00 6B`（18 字节），字段值为 Nearest、CW、30 RPM、10000 ms、300 RPM、800 mA、60 ms、关闭上电自动回零。
- 错误：`01 00 EE 6B`（4 字节）。
- 待验证：手册印刷的同一响应示例在碰撞检测转速位置写成 `0F A0`，但紧邻的字段解析和修改示例明确写成 `0x012C=300 RPM`；本文按字段解析整理为 `01 2C`，原始手册字节矛盾需实测确认。

#### 6.2.5 修改回零参数（`0x4C`）

- 请求：`A 4C AE STORE MODE DIR VEL[2] TIMEOUT[4] SL_RPM[2] SL_MA[2] SL_MS[2] POWER_ON C`（20 字节）。
- 响应：`A 4C ST C`（4 字节）。
- 字段：字段顺序、长度和单位与 `0x22` 响应相同；`STORE` 为存储标志。
- 示例：`01 4C AE 01 00 00 00 1E 00 00 27 10 01 2C 03 20 00 3C 00 6B`（20 字节）→ `01 4C 02 6B`（4 字节）。
- 错误：`01 00 EE 6B`（4 字节）。

#### 6.2.6 读取回零状态（`0x3B`）

- 请求：`A 3B C`（3 字节）。
- 响应：`A 3B HOME_FLAGS C`（4 字节）。
- 字段：`HOME_FLAGS` 1 字节，按第 4.4 节的 bit0–bit3 解释。
- 示例：`01 3B 6B`（3 字节）→ `01 3B 03 6B`（4 字节），表示编码器和校准表就绪、当前未回零且未回零失败。
- 错误：`01 00 EE 6B`（4 字节）。

### 6.3 触发动作命令（手册 6.3.3）

#### 6.3.1 触发编码器校准（`0x06`）

- 请求：`A 06 45 C`（4 字节）。`45` 为固定子命令。
- 响应：`A 06 ST C`（4 字节）。
- 字段：无单位；功能对应屏幕 `Cal` 菜单。
- 示例：`01 06 45 6B`（4 字节）→ `01 06 02 6B`（4 字节）。
- 错误：开环模式或堵转保护触发时 `01 06 E2 6B`；错误命令 `01 00 EE 6B`。

#### 6.3.2 当前的位置角度清零（`0x0A`）

- 请求：`A 0A 6D C`（4 字节）。
- 响应：`A 0A ST C`（4 字节）。
- 字段：`6D` 为固定子命令；无单位。手册说明当前位置角度、位置误差、脉冲数等同时清零。
- 示例：`01 0A 6D 6B`（4 字节）→ `01 0A 02 6B`（4 字节）。
- 错误：`01 00 EE 6B`（4 字节）。

#### 6.3.3 解除堵转保护（`0x0E`）

- 请求：`A 0E 52 C`（4 字节）。
- 响应：`A 0E ST C`（4 字节）。
- 字段：`52` 为固定子命令；无单位。
- 示例：`01 0E 52 6B`（4 字节）→ `01 0E 02 6B`（4 字节）。
- 错误：尚未触发堵转保护时 `01 0E E2 6B`；错误命令 `01 00 EE 6B`。

#### 6.3.4 恢复出厂设置（`0x0F`）

- 请求：`A 0F 5F C`（4 字节）。
- 响应：`A 0F ST C`（4 字节）。
- 字段：`5F` 为固定子命令；无单位。恢复后需要断电重新上电并重新空载校准编码器。
- 示例：`01 0F 5F 6B`（4 字节）→ `01 0F 02 6B`（4 字节）。
- 错误：`01 00 EE 6B`（4 字节）。

## 7. 读取命令

本节覆盖手册 6.3.4 的全部读取命令。除特别说明外，读取请求均为 3 字节 `A F C`，错误响应为 `A 00 EE C`（4 字节）。

### 7.1 固件和硬件版本（`0x1F`）

- 请求：`A 1F C`（3 字节）。
- 响应：`A 1F FW HW C`（5 字节）。`FW`、`HW` 各 1 字节，单位/编码为版本号字段。
- 示例：`01 1F 6B`（3 字节）→ `01 1F 7D 6F 6B`（5 字节）。
- 错误：`01 00 EE 6B`（4 字节）。
- 待验证：手册随后把版本解释为 `FW=0xF4`（Emm42_V5.0.0）、`HW=0x78`（120，ZDT_X42_V1.2），与前一行示例返回的 `7D 6F` 不一致。本文只把 `FW/HW` 记录为原始字节，不推断稳定的版本编码映射。

### 7.2 相电阻和相电感（`0x20`）

- 请求：`A 20 C`（3 字节）。
- 响应：`A 20 R[2] L[2] C`（7 字节）。`R` 单位 mΩ，`L` 单位 µH，均为大端无符号。
- 示例：`01 20 6B`（3 字节）→ `01 20 04 7A 0D 28 6B`（7 字节），即 `R=1146 mΩ`、`L=3368 µH`。
- 错误：`01 00 EE 6B`（4 字节）。

### 7.3 位置环 PID（`0x21`）

- 请求：`A 21 C`（3 字节）。
- 响应：`A 21 KP[4] KI[4] KD[4] C`（15 字节）。三个参数均为大端 32 位无符号数；手册未给出额外物理单位或定点比例。
- 示例：`01 21 6B`（3 字节）→ `01 21 00 00 F2 30 00 00 00 64 00 00 F2 30 6B`（15 字节），即 `Kp=62000`、`Ki=100`、`Kd=62000`。
- 错误：`01 00 EE 6B`（4 字节）。
- 限制：手册只公开位置环 PID 的读写接口。本文不推断速度环或电流环 PID 的寄存器、功能码、地址或数据格式。

### 7.4 总线电压（`0x24`）

- 请求：`A 24 C`（3 字节）。
- 响应：`A 24 V[2] C`（5 字节）。`V` 为大端无符号，单位 mV。
- 示例：`01 24 6B`（3 字节）→ `01 24 5C 6A 6B`（5 字节），即 `23658 mV`。
- 错误：`01 00 EE 6B`（4 字节）。

### 7.5 相电流（`0x27`）

- 请求：`A 27 C`（3 字节）。
- 响应：`A 27 I[2] C`（5 字节）。`I` 为大端无符号，单位 mA。
- 示例：`01 27 6B`（3 字节）→ `01 27 02 73 6B`（5 字节），即 `627 mA`。
- 错误：`01 00 EE 6B`（4 字节）。

### 7.6 线性化校准后的编码器值（`0x31`）

- 请求：`A 31 C`（3 字节）。
- 响应：`A 31 ENC[2] C`（5 字节）。`ENC` 为大端无符号，范围按手册为 `0..65535`，一圈 65536 个计数。
- 示例：`01 31 6B`（3 字节）→ `01 31 8D 9E 6B`（5 字节），即 `ENC=36254`。
- 错误：`01 00 EE 6B`（4 字节）。

### 7.7 输入脉冲数（`0x32`）

- 请求：`A 32 C`（3 字节）。
- 响应：`A 32 SIGN PULSES[4] C`（8 字节）。`SIGN=00` 正、`01` 负；`PULSES` 为大端 32 位幅值，单位为脉冲个数。
- 示例：`01 32 6B`（3 字节）→ `01 32 01 00 00 0C 80 6B`（8 字节），即 `-0x0C80=-3200` 个脉冲。
- 错误：`01 00 EE 6B`（4 字节）。

### 7.8 电机目标位置（`0x33`）

- 请求：`A 33 C`（3 字节）。
- 响应：`A 33 SIGN POS[4] C`（8 字节）。`POS` 是大端 32 位角度计数幅值，单位为 `65536 计数/圈`。
- 示例：`01 33 6B`（3 字节）→ `01 33 01 00 01 00 00 6B`（8 字节），即 `-0x00010000=-360.0°`。
- 错误：`01 00 EE 6B`（4 字节）。

### 7.9 实时设定目标位置/开环实时位置（`0x34`）

- 请求：`A 34 C`（3 字节）。
- 响应：`A 34 SIGN POS[4] C`（8 字节）。在开环模式下，该字段是手册所称的实时位置；单位和角度换算与 `0x33` 相同。
- 示例：`01 34 6B`（3 字节）→ `01 34 01 00 01 00 00 6B`（8 字节），即 `-360.0°`。
- 错误：`01 00 EE 6B`（4 字节）。

### 7.10 实时转速（`0x35`）

- 请求：`A 35 C`（3 字节）。
- 响应：`A 35 SIGN RPM[2] C`（6 字节）。`RPM` 为大端 16 位幅值，单位 RPM；`SIGN` 为独立符号字节。
- 示例：`01 35 6B`（3 字节）→ `01 35 01 05 DC 6B`（6 字节），即 `-1500 RPM`。
- 错误：`01 00 EE 6B`（4 字节）。

### 7.11 实时位置（`0x36`）

- 请求：`A 36 C`（3 字节）。
- 响应：`A 36 SIGN POS[4] C`（8 字节）。角度换算为 `SIGN × POS × 360/65536`。
- 示例：`01 36 6B`（3 字节）→ `01 36 01 00 01 00 00 6B`（8 字节），即 `-360.0°`。
- 错误：`01 00 EE 6B`（4 字节）。

### 7.12 位置误差（`0x37`）

- 请求：`A 37 C`（3 字节）。
- 响应：`A 37 SIGN ERR[4] C`（8 字节）。`ERR` 为大端 32 位幅值，角度单位由 `ERR×360/65536` 换算。
- 示例：`01 37 6B`（3 字节）→ `01 37 01 00 00 00 08 6B`（8 字节），即约 `-0.0439453125°`。
- 错误：`01 00 EE 6B`（4 字节）。

### 7.13 电机状态标志（`0x3A`）

- 请求：`A 3A C`（3 字节）。
- 响应：`A 3A MOTOR_FLAGS C`（4 字节）。标志位见第 4.4 节。
- 示例：`01 3A 6B`（3 字节）→ `01 3A 03 6B`（4 字节），bit0 和 bit1 为 1，表示使能且到位。
- 错误：`01 00 EE 6B`（4 字节）。

### 7.14 驱动配置参数（`0x42`）

- 请求：`A 42 6C C`（4 字节）。`6C` 为固定读取子命令。
- 响应：`A 42 LEN COUNT CONFIG C`（33 字节）。手册示例中 `LEN=0x21` 表示完整响应 33 字节，`COUNT=0x15` 表示 21 个配置参数；`CONFIG` 共 28 字节，由 21 个逻辑字段组成。

  | 5 | 1 | 脉冲端口模式 | 示例 `0x02=PUL_FOC`。其它序号未在此字段表完整定义。 |

  | 9 | 1 | 细分 | `0x10=16`；特殊约定 `0x00=256`。 |

  | 11 | 1 | 自动熄屏 | `0x00=Disable`。 |
  | 12 | 2 | 开环工作电流 | `0x03E8=1000 mA`。 |
  | 14 | 2 | 闭环堵转最大电流 | `0x0BB8=3000 mA`。 |

  | 偏移（串口逻辑） | 长度 | 字段 | 示例值/单位 |
  | ---: | ---: | --- | --- |
  | 2 | 1 | `LEN` | `0x1F`，完整响应长度 31 字节。 |
  | 3 | 1 | `COUNT` | `0x09`，状态参数个数 9。 |
  | 4 | 2 | 总线电压 | `0x5C67=23655 mV`。 |
  | 6 | 2 | 总线相电流 | `0x0003=3 mA`。 |
  | 8 | 2 | 校准后编码器值 | `0x43EB=17387`。 |
  | 10 | 1 | 目标位置符号 | `0x01=负`。 |
  | 11 | 4 | 电机目标位置 | `0x00010000`，合计 `-360°`。 |
  | 15 | 1 | 实时转速符号 | `0x00=正`。 |
  | 16 | 2 | 电机实时转速 | `0x0000=0 RPM`。 |
  | 18 | 1 | 实时位置符号 | `0x01=负`。 |
  | 19 | 4 | 电机实时位置 | `0x00010000`，合计 `-360°`。 |
  | 23 | 1 | 位置误差符号 | `0x01=负`。 |
  | 24 | 4 | 电机位置误差 | `0x00000008`，约 `-0.044°`。 |
  | 28 | 1 | 就绪状态标志 | `0x03`，bit0/bit1 就绪。 |
  | 29 | 1 | 电机状态标志 | `0x03`，bit0 使能、bit1 到位。 |
  | 30 | 1 | 校验 | `0x6B`。 |

- 错误：`01 00 EE 6B`（4 字节）。
- 限制：上表按手册给出的字节顺序记录 9 个状态参数；没有对手册未定义的保留字节或其他状态含义进行扩展。

## 8. 修改命令

本节覆盖手册 6.3.5 的全部修改命令。除特别说明外，成功响应为 `A F 02 C`（4 字节），错误响应为 `A 00 EE C`（4 字节）。

### 8.1 修改任意细分（`0x84`）

- 请求：`A 84 8A STORE MICROSTEP C`（6 字节）。
- 响应：`A 84 ST C`（4 字节）。`MICROSTEP` 为 1 字节；`0x00=256`，其它值按手册允许的任意细分值解释。
- 示例：`01 84 8A 01 07 6B`（6 字节）→ `01 84 02 6B`（4 字节），保存为 7 细分。
- 错误：`01 00 EE 6B`（4 字节）。

### 8.2 修改 ID 地址（`0xAE`）

- 请求：`A AE 4B STORE NEW_ADDR C`（6 字节）。`NEW_ADDR` 范围为 `1..255`；`0` 是广播地址，不应作为普通节点地址保存。
- 响应：`A AE ST C`（4 字节）。
- 示例：`01 AE 4B 01 10 6B`（6 字节）→ `01 AE 02 6B`（4 字节），保存为地址 16。修改生效后后续通讯应使用新地址。
- 错误：`01 00 EE 6B`（4 字节）。

### 8.3 切换开环/闭环模式（`0x46`）

- 请求：`A 46 69 STORE MODE C`（6 字节）。
- 响应：`A 46 ST C`（4 字节）。协议字段中手册明确给出 `MODE=01` 开环、`MODE=02` 闭环；这两个编码不要与 `0x42` 读取字段的菜单序号混用。
- 示例：`01 46 69 01 01 6B`（6 字节）→ `01 46 02 6B`（4 字节）切换为开环；将最后的模式字节改为 `02` 可切换为闭环。
- 错误：`01 00 EE 6B`（4 字节）。

### 8.4 修改开环工作电流（`0x44`）

- 请求：`A 44 33 STORE CURRENT[2] C`（7 字节）。`CURRENT` 为大端无符号，单位 mA。
- 响应：`A 44 ST C`（4 字节）。
- 示例：`01 44 33 00 03 E8 6B`（7 字节）→ `01 44 02 6B`（4 字节），设置为 `1000 mA` 且不保存。
- 错误：`01 00 EE 6B`（4 字节）。

### 8.5 修改驱动配置参数（`0x48`）

- 请求：`A 48 D1 STORE CONFIG C`（33 字节）。`D1` 为固定子命令，`CONFIG` 的 21 个逻辑字段顺序与 `0x42` 响应的偏移 4–31 相同，但请求不包含 `LEN` 和 `COUNT` 两个响应头字段。
- 响应：`A 48 ST C`（4 字节）。
- 示例（33 字节）：

  ```text
  01 48 D1 01 19 02 02 02 00 10 01 00 03 E8 0B B8 0F A0 05 07 01 00 01 01 00 28 09 60 0F A0 00 01 6B
  ```

  示例设置 1.8°电机、PUL_FOC、UART_FUN、En Hold、CW、16 细分、细分插补、开环 1000 mA、闭环堵转最大电流 3000 mA、最大输出电压 4000 mV、串口 115200、CAN 500000、地址 1、固定 0x6B、Receive、堵转保护、40 RPM/2400 mA/4000 ms 阈值和 0.1° 到位窗口。

- 响应示例：`01 48 02 6B`（4 字节）。
- 错误：`01 00 EE 6B`（4 字节）。
- 限制：手册说明该命令的原 ID 字段被取消，使广播地址可以批量修改驱动参数；批量修改后各节点的地址、校验和响应设置需要逐节点核验。

### 8.6 修改位置环 PID（`0x4A`）

- 请求：`A 4A C3 STORE KP[4] KI[4] KD[4] C`（17 字节）。三个 PID 参数均为大端 32 位；手册没有给出比例单位或内部缩放。
- 响应：`A 4A ST C`（4 字节）。
- 示例：`01 4A C3 00 00 00 F2 30 00 00 00 64 00 00 F2 30 6B`（17 字节）→ `01 4A 02 6B`（4 字节），不保存 `Kp=62000`、`Ki=100`、`Kd=62000`。
- 错误：`01 00 EE 6B`（4 字节）。
- 限制：该命令只修改位置环 PID。不能从它推断速度环或电流环 PID 的寄存器。

### 8.7 存储速度模式上电参数（`0xF7`）

- 请求：`A F7 1C STORE DIR RPM[2] ACC EN_CTRL C`（10 字节）。`1C` 为固定子命令；`EN_CTRL=00` 不使用 En 控制启停，`01` 使用 En 控制启停。
- 响应：`A F7 ST C`（4 字节）。
- 字段：`DIR=00` CW、`01` CCW；`RPM` 单位 RPM；`ACC` 为加速度档位；`STORE` 为存储/清除标志；`EN_CTRL` 为 En 控制标志。
- 示例：`01 F7 1C 01 01 05 DC 0A 01 6B`（10 字节）→ `01 F7 02 6B`（4 字节），保存 CCW、1500 RPM、加速度 10、启用 En 控制的上电参数。
- 错误：`01 00 EE 6B`（4 字节）。

### 8.8 通讯速度输入缩小 10 倍（`0x4F`）

- 请求：`A 4F 71 STORE S_VEL_IS C`（6 字节）。`S_VEL_IS=00` 不使能，`01` 使能。
- 响应：`A 4F ST C`（4 字节）。
- 字段：无物理存储单位；使能后速度模式、位置模式和上电速度参数的通讯输入值都缩小 10 倍。
- 示例：`01 4F 71 01 01 6B`（6 字节）→ `01 4F 02 6B`（4 字节）。
- 错误：`01 00 EE 6B`（4 字节）。

## 9. 完整报文示例

以下示例专门对照串口逻辑报文、CAN ID 和 CAN payload。所有示例采用固定校验 `0x6B`。

### 9.1 读取总线电压

```text
串口请求（3 字节）：01 24 6B
串口响应（5 字节）：01 24 5C 6A 6B       # 0x5C6A=23658 mV

CAN 请求：ID=0x0100，payload（2 字节）：24 6B
CAN 响应：ID=0x0100，payload（4 字节）：24 5C 6A 6B
```

### 9.2 读取电机状态

```text
串口请求（3 字节）：01 3A 6B
串口响应（4 字节）：01 3A 03 6B          # 使能且到位

CAN 请求：ID=0x0100，payload（2 字节）：3A 6B
CAN 响应：ID=0x0100，payload（3 字节）：3A 03 6B
```

### 9.3 读取位置环 PID

```text
串口请求（3 字节）：01 21 6B
串口响应（15 字节）：01 21 00 00 F2 30 00 00 00 64 00 00 F2 30 6B
```

`0x21` 的响应超过单个 CAN payload 的 8 字节。手册只给出 CAN 长请求的分包示例，没有给出该长响应的 CAN 分包响应实例；不要仅凭串口响应推断 CAN 接收分包格式。

### 9.4 修改位置环 PID

```text
串口请求（17 字节）：01 4A C3 00 00 00 F2 30 00 00 00 64 00 00 F2 30 6B
串口响应（4 字节）： 01 4A 02 6B
```

### 9.5 大于 8 字节的位置命令分包

```text
串口逻辑请求（13 字节）：01 FD 01 0F A0 00 00 01 FA 00 00 00 6B

CAN 第 0 包：ID=0x0100，payload（8 字节）：FD 01 0F A0 00 00 01 FA
CAN 第 1 包：ID=0x0101，payload（5 字节）：FD 00 00 00 6B
CAN 成功响应：ID=0x0100，payload（3 字节）：FD 02 6B
```

这里串口命令的地址 `01` 没有出现在两个 CAN payload 中，功能码 `FD` 在两个包中都出现，校验 `6B` 只在最后一包出现。

### 9.6 广播速度命令

```text
串口广播请求（8 字节）：00 F6 00 05 DC 0A 00 6B
```

该命令要求所有节点以 CW、1500 RPM、加速度 10 运行；所有地址节点执行，只有地址 1 返回：

```text
地址 1 响应（4 字节）：01 F6 02 6B
地址 2、3、...：不回复
```

CAN 广播请求对应 `CAN_ID=0x0000`（第 0 包）、payload=`F6 00 05 DC 0A 00 6B`；若地址 1 回复，响应帧应以地址 1 的 ID 承载地址，即 `ID=0x0100`、payload=`F6 02 6B`。广播 CAN 的仲裁、多个节点同时执行以及回复冲突行为仍应按实际总线测试。

### 9.7 成功、条件失败和通用错误

以速度功能码 `F6` 为例：

```text
成功：       01 F6 02 6B
条件不满足： 01 F6 E2 6B
通用错误：   01 00 EE 6B
```

在地址 1、CAN 第 0 包上，对应 payload 分别为：

```text
成功：       F6 02 6B
条件不满足： F6 E2 6B
通用错误：   00 EE 6B
```

## 11. Klipper 位置误差实时监控

仓库中的 `[zdt_emm42 <name>]` Klipper extra 使用 Linux SocketCAN 读取 EMM42 的运行参数。正常监控时运动仍由 Klipper 的 STEP/DIR/EN 控制，插件只发送短读取命令；显式执行第 12 节的自动调参命令时，插件才会通过 Klipper toolhead 执行用户确认的测试运动并发送 PID 写入命令。

### 11.1 `0x37` 独立采样

位置误差请求必须保持协议规定的 CAN 形态：

```text
串口逻辑请求：01 37 6B
CAN 扩展帧 ID：0x0100       # ID_Addr=1，packet_number=0
CAN payload：37 6B
```

插件将 `0x37` 从普通遥测轮询中拆出，使用 `error_poll_interval` 独立采样，默认值为 `0.10` 秒。`poll_interval` 仍控制其它读取命令；两类请求可以各有一个短暂的异步响应等待，不会把位置误差降低到完整轮询列表的周期。

### 11.2 位置误差状态字段

`zdt_emm42` 对象的状态至少包含以下曲线所需字段：

| 字段 | 含义 |
| --- | --- |
| `error_deg` | 最近一次有效 `0x37` 响应的位置误差角度。 |
| `error_counts` | 最近一次有效响应的带符号误差计数。 |
| `last_update_time` | 最近一次有效误差样本的 Klipper reactor 单调时间。 |
| `online` | 在 `offline_timeout` 内收到有效 `0x37` 响应且轮询已启用时为 `true`。 |
| `error_count` | 超时、发送失败、通用错误和校验失败的累计计数。 |
| `error_history` | 最近 5 秒有效样本列表，每项包含 `time`、`error_deg` 和 `error_counts`。 |

误差角度必须按协议的符号与幅值规则计算：

```text
error_deg = sign × raw_value × 360 / 65536
```

`0x00` 为正、`0x01` 为负；`raw_value` 是 32 位大端无符号幅值，不应按补码解释。协议示例 `sign=0x01`、`raw_value=0x00000008` 的结果是约 `-0.0439453125°`。

历史窗口由有效样本时间戳裁剪，并设置固定的最大队列长度，始终不会无限增长。超时、错误响应、错误校验或设备离线不会写入零值样本；前端应使用 `online` 判断 CAN 状态，而不是把失联当成误差为零。

### 11.3 Mainsail Dashboard 面板

配套的 `mainsail-toolchanger` 会自动识别单个 `[zdt_emm42 <name>]` 对象，并在 Dashboard 提供可折叠、可排序且持久化布局的“EMM42 位置误差”面板。面板使用 `error_history` 绘制最近 5 秒滚动曲线，横轴为相对秒数，纵轴为角度 `°`，零线单独标出，同时显示当前值、5 秒最大绝对误差、采样周期和 CAN 在线状态。

未配置 `zdt_emm42` 时面板显示配置提示；配置存在但没有有效响应时显示离线提示，并且不绘制伪造零值。曲线窗口与 CSV 日志相互独立：曲线始终只看最近 5 秒，而 CSV 仍由 `csv_path`、`ZDT_EMM_LOG ENABLE=1` 和 `ZDT_EMM_LOG ENABLE=0` 控制。

## 12. Klipper 在线 PID 自动调参

仓库中的 Klipper extra 提供一个显式运动测试命令，只调整 EMM42 的位置环 PID（`0x21`/`0x4A`），不修改 `0x48` 驱动配置：

```gcode
ZDT_EMM_AUTOTUNE NAME=shadow_a AXIS=X DISTANCE=10 SPEED=20 ACCEL=200 ITERATIONS=20 CONFIRM=1
```

命令要求打印机空闲、指定轴（X/Y/Z）已经回零、CAN 设备在线，并要求 `CONFIRM=1`。`E` 轴可以作为显式测试轴使用，但不具备 X/Y/Z 的回零状态检查。`DISTANCE` 是单方向往返距离；每轮执行“当前位置 → 正方向距离 → 原位置”的运动，并等待 `autotune_settle_time` 后再结束本轮采样。

### 12.1 CAN PID 读写流程

自动调参只在以下配置下运行：

```ini
can_payload_includes_addr: False
```

地址必须放在扩展 CAN ID，不能重复放进 payload。命令开始时先读取 `0x21` 当前 Kp、Ki、Kd，然后用相同参数发送一次 `0x4A`、`STORE=0` 的临时写入，并再次读取 `0x21` 校验。验证失败会在任何运动发生前终止。

`0x4A` 的 CAN 长请求沿用第 3.3 节的分包规则。以位置环 PID 为例，串口逻辑报文为：

```text
01 4A C3 STORE KP[4] KI[4] KD[4] 6B
```

地址从 payload 移到 CAN ID 后，每个 CAN payload 都重复 `4A`，每包最多携带 7 个后续字节，校验字节只放在最后一包。`0x21` 的 15 字节响应也按相同的“功能码重复、CAN ID 低字节为包号”规则重组；如果设备返回乱序、长度错误、超时、错误响应或校验失败，读写均视为失败。

### 12.2 单轴兼容模式的搜索和评分

未指定 `PROFILE` 或显式使用 `PROFILE=AXIS` 时，`ITERATIONS` 表示在线搜索轮数。基准值测试完成后，插件按 Kp、Ki、Kd 循环，每轮只改变一个 PID 参数，使用当前步长和方向生成下一候选值，并立即用 `STORE=0` 写入和回读。运动结束后仅使用该轮时间范围内新产生的 `error_history` 样本计算评分；候选改善评分则保留，否则恢复最后一组有效 PID、反转方向，连续失败后减小步长。

评分综合以下指标：

- 误差 RMS；
- 最大绝对误差；
- 误差过零后的超调；
- 进入误差阈值的稳定时间；
- 过零次数、CAN 错误、离线和电机堵转会造成拒绝或惩罚。

默认步长可以在 `[zdt_emm42]` 中设置：`autotune_kp_step`、`autotune_ki_step`、`autotune_kd_step`；单次命令也可以用 `KP_STEP`、`KI_STEP`、`KD_STEP` 覆盖。`autotune_min_samples` 控制每轮最少有效误差样本数，`pid_write_settle_time` 控制 `0x4A` 响应后等待驱动器内部参数生效的时间。每次回读最多重试三次，避免驱动器确认写入但尚未立即反映到 `0x21` 读取结果时中断搜索。

搜索结束后，评分最佳值使用 `0x4A STORE=1` 直接写入驱动器内部存储，并回读确认。该写入不修改 `printer.cfg`，不要求重启 Klipper；如果运动、CAN、回读或最终持久化失败，插件会尽力把原始 PID 写回运行时，并输出失败原因。

### 12.3 CoreXY 打印工况模式

单轴兼容模式只评价一组固定往返运动。CoreXY 打印机应使用：

```gcode
ZDT_EMM_AUTOTUNE NAME=shadow_a PROFILE=COREXY_PRINT DISTANCE=10 SPEED=200 ACCEL=5000 ITERATIONS=20 REPEATS=3 MAX_ERROR_DEG=5 CONFIRM=1
```

`SPEED` 和 `ACCEL` 是实际切片最高打印参数。插件自动生成 40%、70%、100% 三档测试，轨迹位于当前位置的正 X/正 Y 方形区域，覆盖 X、Y、`X+Y`、`X-Y` 和连续拐角。X/Y 必须均已回零，且正方向必须具有 `DISTANCE` 指定的安全空间。

CoreXY 模式在每个 PID 候选下重复每档工况，默认 `REPEATS=3`，使用中位数降低机械噪声和 CAN 采样相位的影响。调参期间暂停普通遥测轮询，`0x37` 使用独立 20 ms 采样周期和独立内存缓冲；因此测试超过 5 秒时也不会被 Dashboard 历史窗口裁掉。

每个样本按运动和停止稳定阶段标记，正常方向变化不再作为超调。单次评分为：

```text
score = 0.50 × motion_rms
      + 0.30 × motion_p95
      + 0.15 × motion_peak
      + 0.05 × settle_rms
```

三档工况等权平均。候选分数至少改善 `MIN_IMPROVEMENT`（默认 `0.02`，即 2%）才被接受。每次迭代对当前 Kp、Ki 或 Kd 同时评价正负两个步长；完成 Kp/Ki/Kd 一轮后步长减半，连续两轮无显著改善时提前收敛。

Kp/Kd 默认边界为原始值的 `0.5–2.0` 倍，Ki 默认边界为 `0–max(原始值×10, 原始值+10×KI_STEP)`。命令可使用 `KP_MIN/KP_MAX`、`KI_MIN/KI_MAX`、`KD_MIN/KD_MAX` 覆盖，但边界必须包含原始值。

搜索完成后，插件分别用原始值和候选最佳值运行未参与搜索的 55%、85% 档及不同顺序轨迹。验证改善不足 2% 时恢复原始 PID，不写入芯片；验证通过后才执行 `STORE=1`。`MAX_ERROR_DEG` 是必填硬门槛，误差越线会拒绝候选，CAN 离线或堵转会终止调参。`ZDT_EMM_AUTOTUNE_CANCEL NAME=<name>` 可请求取消，当前运动段完成后恢复参数。
